## 落地建议分类

### 推荐立即应用（高价值、低风险）

这些特性与本项目（Streamlit 物理实验答题系统）直接相关，接入成本低、收益明显。

#### 1. `st.pagination` 分页（1.58.0）
- **应用场景**：错题本、排行榜、题库浏览列表。
- **现状**：当前错题本/排行榜使用循环展开全部数据，题目或用户量增多后页面冗长。
- **建议**：将 `for entry in filtered:` 改为 `paginated = st.pagination(filtered, items_per_page=10)`，大幅简化分页逻辑并提升体验。
- **参考文件**：[v1.58.0](v1.58.0.md)

#### 2. `st.status` 加载/保存状态（1.48.0 新设计，1.58.0 支持 `type`）
- **应用场景**：题库加载、存档保存、localStorage 读写反馈。
- **现状**：加载过程缺少可视化状态提示。
- **建议**：在答题练习页/载入题库页用 `with st.status("正在加载题库...", expanded=True) as status:` 包装加载流程；完成时调用 `status.update(label="加载完成", state="complete", expanded=False)`。1.58.0 还可使用 `type="compact"` 减少占用空间。
- **参考文件**：[v1.58.0](v1.58.0.md)、[v1.48.0](v1.48.0.md)

#### 3. `@st.fragment(parallel=True)` 局部并行刷新（1.58.0）
- **应用场景**：排行榜刷新、localStorage 后台同步、计时器。
- **现状**：`utils/localstorage.py` 使用同步 `components.html` 注入，且全页 rerun 成本较高。
- **建议**：将 localStorage 读写、排行榜数据刷新等独立任务封装为 `@st.fragment(parallel=True)` 函数，减少整页重渲染，提升并发体验。
- **参考文件**：[v1.58.0](v1.58.0.md)

#### 4. `st.segmented_control` / `st.pills` 替代单选/多选（1.40.0）
- **应用场景**：题型筛选（单选/多选/判断）、难度选择、章节快速切换。
- **现状**：使用 `st.radio` 或 `st.selectbox` 占用垂直空间较多。
- **建议**：在错题本筛选、智能分析维度选择等处使用 `st.segmented_control` 或 `st.pills`，节省空间并提升视觉层次。
- **参考文件**：[v1.40.0](v1.40.0.md)

#### 5. `st.toast(duration=...)` 轻量提示（1.49.0）
- **应用场景**：保存成功、答题提交、错题移除提示。
- **现状**：可能使用 `st.success` 等占用主页面区域的提示。
- **建议**：将非阻塞反馈改为 `st.toast("已保存", duration=2.0)`，避免打断答题流程。
- **参考文件**：[v1.49.0](v1.49.0.md)

#### 6. `st.feedback` 题目难度评分（1.37.0）
- **应用场景**：答完题后让用户对题目难度打星。
- **建议**：在答题练习页提交答案后，使用 `st.feedback("stars", key=f"feedback_{qid}")` 收集反馈，用于后续推荐算法。
- **参考文件**：[v1.37.0](v1.37.0.md)

## 对当前答题系统的 5 条最重要落地建议

1. **用 `st.pagination` 重构错题本与排行榜分页**
   - 目标文件：`pages/03_错题本.py`、`pages/05_排行榜.py`
   - 将当前全量循环展示改为分页展示，降低长列表渲染压力。

2. **用 `st.status` 包装题库加载与存档保存**
   - 目标文件：`pages/04_载入题库.py`、`utils/storage.py`
   - 提供清晰的“加载中 / 完成 / 错误”状态反馈，提升用户感知。

3. **用 `@st.fragment(parallel=True)` 优化 localStorage 与排行榜刷新**
   - 目标文件：`utils/localstorage.py`、`pages/05_排行榜.py`
   - 将耗时或独立任务隔离为并行 fragment，避免全页 rerun。

4. **用 `st.segmented_control` / `st.pills` 替换部分筛选器**
   - 目标文件：`pages/03_错题本.py`、`pages/02_智能分析.py`
   - 将 `st.radio` 筛选题型/难度改为按钮组，提升交互效率。

5. **用 `st.toast` 替代部分阻塞式成功提示**
   - 目标文件：`pages/01_答题练习.py`、`pages/03_错题本.py`
   - 保存、提交、移除错题后使用短时 toast，不打断答题流程。
