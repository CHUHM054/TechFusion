# 新题型参考目录说明

本目录用于存放各类新题型的详细解题流程示例，供生成题目与解析时参考。

## 文件命名规范

每个示例文件统一命名为：

```
<章节名>——<方法名>.md
```

例如：

- `无限长序列和有限长序列卷积——重叠相加法和重叠保留法.md`
- `离散傅里叶变换——按时间抽选基2FFT.md`

## 如何填写 TEMPLATE.md

参考本目录下的 `TEMPLATE.md`，按以下六个章节填写：

1. **# 题目**：给出已知序列/信号、参数和待求目标。
2. **# 待求目标**：用条目清晰列出考生需要求出的内容。
3. **# 符号表**：用表格列出符号、中文含义、类型和 KaTeX 公式。类型请从 `scalar`、`sequence`、`index_range`、`dict` 中选择。
4. **# Step 列表**：每个步骤使用 `### StepN：标题` 形式，写出详细推导/计算流程。
5. **# 挖空建议**：标注希望考生填写的位置，类型可以是 `单值`、`列表`、`字典`。
6. **# 参考答案**：给出标准答案或关键结果，便于校验。

填写时请注意：

- 数学公式统一使用 KaTeX 语法，并使用 Streamlit 兼容的定界符：行内 `$...$`，块级 `$$...$$`。
- 序列用 `$\{a, b, c\}$` 表示，带索引范围可写作 `$\{...\}_{[0, N-1]}$`。
- 符号表中的 `formula` 字段必须能被 Streamlit 原生渲染，统一用 `$...$` 包裹。
- 保持步骤清晰、公式完整，便于后续解析生成题目。

## 现有示例

## Agent 读取与生成流程

Agent 会按以下步骤读取本目录下的参考文件并生成题目草稿：

1. **读取 TEMPLATE.md**：识别六个标准章节和符号表结构。
2. **读取具体示例文件**：根据章节名和方法名匹配目标题型。
3. **解析符号表**：提取每个符号的类型、含义和公式，用于生成 `calc/*.json` 中的变量定义。
4. **解析 Step 列表**：将每个步骤拆分为输入条件、计算过程和输出结果，生成对应的计算节点。
5. **解析挖空建议**：根据 `单值`、`列表`、`字典` 类型，在题目中生成对应的填空项。
6. **生成草稿**：输出 `calc/*.json` 草稿，包含题目文本、符号定义、步骤节点、挖空位置和参考答案。

生成后建议人工检查公式、索引范围和最终答案，确保与参考文件一致。
